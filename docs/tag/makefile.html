Local node

Almost all the tools used for our experiment requires Python modules. However, it's required to install some Linux tool

Linux

the following are Linux packages required to be already installed in the local host:

  • socat
  • jq
  • Python (2.7 and >=3.5)
  • python-tk
  • python3-tk
  • pip3
  • virtualenvwrapper
  • virtualenv
  • npm
  • tshark

They can be installed by means of apt-get or similar For debian based system they can be installed as follows:

apt-get install jq socat npm
apt-get install python-pip python3-pip
apt-get install python-tk python3-tk virtualenv  virtualenvwrapper
apt-get install tftpd-hpa # Or your favorite TFTP server

Python

Many of the Python modules are still in development (Iotlab-mqtt, ssh-cli-tools) and are better tested in Python 2 than Python 3. However, ota-server, which make use of asyncio, requires Python 3.2 or superior. This brings as consequence that python2 and python3 must be used in a mix, making complex the use of virtual environments.

Therefore, for installing all the pythons modules are suggested to install them manually with the argument --user.

Still, scripts/Makefile can be used for installing the modules (using the --user argument). You can use the following commands: sudo:

make clean
make get-dist
make pip-install-user

Remote node (FIT IoTLab frontend)

The remote node only requires the Iot-lab tools installed. All of them work fine with Python2, therefore, it's possible to use the script/Makefile as follows:

bash make server

Of course, this means that the repository, or at least the Makefile, should be copied in the frontend.

Yet, because there will SSH commands sends from the local node some considerations are required:

  1. Update the ~/.bashr to let Python to be executable without interactive shell.
  2. Adding the remote node ~/.ssh/id_rsa.pub to your FIT IoT-Lab account.
